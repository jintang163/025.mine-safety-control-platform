## 二、核心技术栈

### 2.1 边缘层
- **EdgeX Foundry 3.1**: 边缘计算框架，提供设备接入、数据处理能力
- **设备协议**:
  - Modbus RTU/TCP: 传统工业设备接入
  - OPC UA: 工业自动化标准协议
  - CAN 总线: 井下车辆、设备通信
  - 4G/5G: 无线传输

### 2.2 消息层
- **EMQX 5.7**: MQTT Broker，支持百万级并发接入
- **Kafka 7.6**: 高吞吐量消息队列，数据缓冲和流处理

### 2.3 应用层
- **Spring Boot 3.2.5**: 后端服务框架
- **Java 17**: 开发语言
- **Spring Data JPA**: ORM框架
- **Spring Data Redis**: 缓存操作
- **Spring Kafka**: Kafka集成

### 2.4 数据层
- **MySQL 8.0**: 关系型数据库，存储配置、报警等结构化数据
- **InfluxDB 2.7**: 时序数据库，存储传感器时序数据
- **Redis 7.2**: 缓存、分布式锁、实时数据存储

## 三、数据流设计

### 3.1 传感器数据采集流程
```
传感器 → EdgeX设备服务 → 数据预处理（去噪/异常检测）
    → MQTT (mine/sensor/data) → Spring Boot MQTT Listener
    → Kafka (sensor-raw-data) → 数据处理服务
    → InfluxDB + MySQL → 应用层查询/展示
```

### 3.2 数据格式（MQTT Payload）
```json
{
  "sensorId": "GAS-001",
  "value": 0.85,
  "timestamp": "2024-01-15T10:30:00",
  "location": "回风巷工作面A",
  "coordinatesX": 116.5,
  "coordinatesY": 39.8,
  "coordinatesZ": 520.5,
  "unit": "% CH4",
  "sensorType": "GAS",
  "quality": 1,
  "protocol": "MODBUS_RTU",
  "gatewayId": "EDGE-GW-001"
}
```

### 3.3 采集频率配置
| 传感器类型 | 采集频率 | 说明 |
|-----------|---------|------|
| 瓦斯浓度 | 1次/秒 | 安全重点监测 |
| 粉尘浓度 | 5次/秒 | 可配置 |
| 一氧化碳 | 2次/秒 | 可配置 |
| 温度 | 5次/秒 | 可配置 |
| 风速 | 10次/秒 | 可配置 |

## 四、报警联动机制

### 4.1 报警级别
| 级别 | 颜色 | 响应时间 | 说明 |
|------|------|---------|------|
| EMERGENCY | 红色 | 立即 | 紧急情况，需立即撤离 |
| ALERT | 橙色 | 5分钟内 | 报警状态，需现场处理 |
| WARNING | 黄色 | 30分钟内 | 预警状态，需关注 |
| INFO | 蓝色 | 无 | 信息通知 |

### 4.2 报警规则示例
| 传感器类型 | 预警阈值 | 报警阈值 | 持续时间 |
|-----------|---------|---------|---------|
| 瓦斯(CH4) | ≥0.8% | ≥1.0% | 5秒/3秒 |
| 粉尘 | ≥200mg/m³ | ≥500mg/m³ | 10秒/5秒 |
| CO | ≥24ppm | ≥50ppm | 5秒/3秒 |
| 温度 | ≥26℃ | ≥30℃ | 30秒/10秒 |
| 风速 | ≤0.5m/s | ≤0.3m/s | 30秒 |

### 4.3 报警通知渠道
- **短信通知 (SMS)**: 集成短信网关API
- **邮件通知 (Email)**: SMTP协议
- **语音通知 (Voice)**: 集成语音网关
- **Webhook**: 支持第三方系统对接
- **实时推送**: WebSocket推送到前端

## 五、EdgeX Foundry 边缘网关

### 5.1 核心服务
- **Core Data**: 数据核心服务，存储和转发设备数据
- **Core Metadata**: 元数据管理，设备和设备配置文件管理
- **Core Command**: 命令控制服务，下发控制命令

### 5.2 设备服务
- **device-modbus**: Modbus RTU/TCP 设备接入
- **device-opcua**: OPC UA 设备接入
- **device-mqtt**: MQTT 设备接入（4G/5G无线）
- **device-can**: CAN 总线设备接入

### 5.3 应用服务
- **app-service-configurable**: 数据导出服务，将数据转发到云端MQTT

## 六、部署架构

### 6.1 硬件配置建议

#### 边缘网关（井下）
| 配置项 | 规格 |
|--------|------|
| CPU | 4核 ARM Cortex-A72 或同等性能 |
| 内存 | 8GB RAM |
| 存储 | 64GB SSD（工业级） |
| 网络 | 双千兆网口 + 4G/5G模块 + CAN接口 |
| 防护等级 | IP65以上 |
| 工作温度 | -40℃ ~ 70℃ |

#### 云端/中心服务器
| 配置项 | 规格 |
|--------|------|
| CPU | 16核 Xeon E5 或同等性能 |
| 内存 | 32GB RAM |
| 存储 | 2TB SSD (RAID 1) |
| 网络 | 千兆光纤接入 |
| 操作系统 | Linux (CentOS 7/Ubuntu 20.04) |

### 6.2 容器化部署
所有服务采用 Docker Compose 部署，支持横向扩展。

## 七、安全设计

### 7.1 网络安全
- 井下网络与办公网络物理隔离
- 防火墙规则限制访问
- VPN 远程接入
- SSL/TLS 加密传输

### 7.2 应用安全
- 用户认证与授权
- API 访问令牌
- 操作审计日志
- 数据加密存储

### 7.3 数据安全
- 数据备份策略
- 异地容灾
- 数据脱敏展示
